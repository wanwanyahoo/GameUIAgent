# 项目阶段工作流

## 1. 产品和规格阶段

推荐顺序：

1. `brainstorming`
2. `ai-app-designer`
3. `game-ui-designer`
4. `game-art-director`
5. `git-commit`

产物：

- PRD。
- 功能地图。
- 技术方案。
- 实现路线图。

## 9. 审查和提交阶段

推荐顺序：

1. `bits-code-guard`
2. `test-driven-development`
3. `git-commit`

检查项：

- 是否有未解释的变更。
- 是否包含系统文件。
- 是否有未完成项或模糊条目。
- 是否符合当前阶段路线图。
- 是否需要推送到远程仓库。
